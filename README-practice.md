# 📝 구현 체크리스트

## ✔ 1. 입력

- [ ] 월, 시작 요일 입력 받기 (`숫자,요일`)
- [ ] 잘못된 경우 `[ERROR]` 출력 후 **월+요일부터 다시 입력**

- [ ] 평일 근무 순번 입력 받기
- [ ] 휴일 근무 순번 입력 받기
- [ ] 잘못된 경우 `[ERROR]` 출력 후 **평일 순번부터 다시 입력**

---

## ✔ 2. 달력 / 요일 처리

- [ ] 월에 따른 날짜 수 계산 (2월은 28일)
- [ ] 시작 요일 기준으로 날짜별 요일 계산
- [ ] 주말(토/일) 판별
- [ ] 법정공휴일 판별
- [ ] 평일 + 공휴일이면 `(휴일)` 표기

---

## ✔ 3. 비상 근무 배정

- [ ] 평일은 평일 순번에서 배정
- [ ] 휴일은 휴일 순번에서 배정
- [ ] 순번이 끝나면 다시 처음부터 사용

---

## ✔ 4. 연속 근무 방지

- [ ] 전날 근무자와 같으면 다음 근무자와 순서 교체
- [ ] 교체는 **앞 날짜부터** 적용
- [ ] 평일 순번 / 휴일 순번 각각 독립적으로 관리

---

## ✔ 5. 출력

- [ ] 형식: `5월 5일 금(휴일) 루루`
- [ ] 날짜별로 한 줄씩 출력
- [ ] 출력이 끝나면 프로그램 종료

---

## ✔ 6. 구조 (MVC 패턴)

- [ ] Model — 근무표 생성 로직
- [ ] View — 입력/출력
- [ ] Controller — 흐름 제어
- [ ] 도메인 로직과 UI 분리

---

## ✔ 7. 코드 규칙

- [ ] 들여쓰기 depth 최대 2
- [ ] 함수 길이 15줄 이하
- [ ] else 지양
- [ ] Jest 테스트 통과 확인
- [ ] `process.exit()` 사용 금지
- [ ] 외부 라이브러리 사용 금지

---

# 🚨 예외 처리 목록

모든 예외는 공통 메시지를 사용한다:
[ERROR] 유효하지 않은 입력 값입니다. 다시 입력해 주세요.

---

## ❌ 1. 월 + 요일 입력 예외

- [ ] 콤마 형식이 아님 (예: `5월월`)
- [ ] 값이 비어 있음
- [ ] 월이 숫자가 아님 (예: `a,월`)
- [ ] 유효하지 않은 월 (0, 13 등)
- [ ] 존재하지 않는 요일 (예: `5,강`)

➡ 다시: **월 + 요일부터 재입력**

---

## ❌ 2. 근무 순번 입력 예외

- [ ] 값이 없음
- [ ] 콤마 기준으로 나눈 값이 비어 있음
- [ ] 닉네임 중복
- [ ] 평일/휴일 순번 길이가 다름
- [ ] 평일/휴일 구성원이 서로 다름

➡ 다시: **평일 순번부터 재입력**

---
